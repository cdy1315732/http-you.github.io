# 小红书研究与发布流程技能说明书

## 1. 技能概述

`xiaohongshu-research-publisher` 是一套面向 OpenClaw / Codex 的通用内容生产与发布技能。

它解决的不是“写一篇小红书文案”这么单一的问题，而是一整条可执行链路：

1. 接收一个题材方向
2. 在全网上收集高信号内容
3. 整理成适合小红书的结构
4. 自动生成封面图和内容图
5. 生成标题与正文
6. 接管已登录的小红书创作中心
7. 上传、发布并校验是否成功

这个技能适用于任何题材，不局限于某个固定话题。只要输入的是一个方向，它就能从“研究”走到“发布”。

## 2. 这个技能是用来做什么的

它适合以下任务：

- 围绕一个题材做小红书图文
- 对某个方向做“热门场景 / 热门技巧 / 推荐清单 / 模型对比 / 使用建议”类整理
- 先全网研究，再转成适合小红书的内容结构
- 需要同时生成封面图、内容图、标题、正文并完成发布
- 希望让 OpenClaw 执行完整的小红书发布流程，而不是只产出文案

典型示例：

- “帮我围绕 AI 自动化做一篇小红书，先搜最近火的内容，再出图并发掉。”
- “整理 10 条 OpenClaw 应用场景，做成小红书图文并发布。”
- “我给你一个方向：本地部署模型怎么选。你先研究，再写，再配图，再发布。”

## 3. 适用边界

### 适合

- 需要跨平台搜集资料的选题
- 需要图文结构化输出的小红书内容
- 需要实际发到小红书创作中心的任务
- 需要统一视觉风格的封面图与内容图

### 不适合

- 只需要闲聊，不需要成品输出
- 只需要普通文章，不需要小红书格式
- 只要单张海报，不需要图文页
- 账号未登录、无法接管浏览器、无法发布的环境

## 4. 技能核心能力

这个技能包含 4 类能力：

### 4.1 研究能力

收到题材后，不会只依赖小红书站内搜索，而是优先在全网上交叉验证。

默认优先级：

1. 官方文档 / 官方产品页 / 官方公告
2. GitHub 仓库 / issues / discussions / release notes
3. 知乎、36 氪、公众号文章、高质量博客、教程、厂商指南
4. 小红书站内搜索结果
5. 其他高信号社区帖子、实战复盘、清单文章

研究目标不是搬运某一篇内容，而是抽取多个来源里反复出现的高信号内容，例如：

- 高频应用场景
- 高频痛点
- 高频技巧
- 高频避坑点
- 高频推荐

所有来源会按平台整理进 `research-notes.md`。

### 4.2 内容结构化能力

技能会根据题材自动选择更适合的小红书结构。

常见结构：

- `10 条结构`：适合场景、技巧、推荐、错误清单
- `5 条结构`：适合模型、技能、方案对比
- `3 段结构`：适合 SOP、教程、方法论

默认图文结构：

- 1 张封面图
- 5 张内容图
- 每张内容图 2 个信息点

### 4.3 生图能力

技能内置图卡生成脚本：

- `scripts/generate_xhs_cards.py`

当前默认输出比例：

- `1080 x 1440`
- `3:4` 竖版

这条规则同时适用于：

- 封面图
- 双分格内容图

### 4.4 发布能力

当素材准备完成后，技能会尝试接管已登录的小红书创作中心完成发布。

发布成功的判断不是“按钮点下去就算”，而是必须同时满足：

1. URL 包含 `/publish/success`
2. `POST https://edith.xiaohongshu.com/web_api/sns/v2/note` 返回 `200`

## 5. 技能完整工作流

这项技能的标准执行顺序如下：

1. 接收题材方向
2. 拆解题材和目标
3. 全网研究高信号内容
4. 整理研究笔记
5. 生成标题与正文
6. 生成封面图与内容图
7. 检查小红书登录态
8. 上传 PNG 图文素材
9. 填充标题和富文本正文
10. 点击发布
11. 校验成功页和发帖请求

这也是为什么它是“流程型技能”，而不是单纯的“文案技能”。

## 6. 输入与输出

### 输入

用户通常只需要提供一个题材方向，例如：

- `OpenClaw 使用技巧`
- `AI 自动化工作流`
- `本地部署模型怎么选`
- `企业自动化场景`
- `个人效率工具`

### 输出

默认会生成一套可发布素材，目录一般类似：

```text
output/xhs-<topic>-<YYYYMMDD>/
```

目录中至少包含：

```text
01-cover-*.svg(.png)
02-*.svg(.png) ~ 06-*.svg(.png)
post.md
research-notes.md
spec.json
style-selection.json
```

说明：

- `post.md`：最终发布文案
- `research-notes.md`：研究来源和整理结论
- `spec.json`：图卡生成说明
- `style-selection.json`：本次随机选中的风格 preset

## 7. 如何触发这个技能

当用户表达以下意图时，应该触发这个技能：

- “给你一个题材，去做一篇小红书并发布”
- “先全网搜这个方向最近比较火的内容，再出图文”
- “做一篇小红书，包含封面图、内容图、标题、正文和发布”
- “围绕这个题材做一整套研究、文案、配图和小红书发布流程”

## 8. 如何使用这个技能

### 8.1 最简用法

直接给题材方向：

```text
围绕 OpenClaw 使用技巧做一篇小红书，先全网搜集热门内容，再生成图文并发布。
```

### 8.2 指定内容结构

```text
围绕 AI 自动化做一篇小红书，整理成 10 条热门应用场景，生成封面图和内容图并发布。
```

### 8.3 指定视觉风格

```text
围绕本地部署模型做一篇小红书，风格固定为 xhs-prd-ref，整篇统一，不要混搭。
```

### 8.4 指定随机风格

```text
围绕企业自动化工作流做一篇小红书，在 editorial-airy 和 xhs-prd-ref 两套风格里随机选一套，整篇统一。
```

## 9. 视觉风格系统

当前技能内置两套 preset：

### `editorial-airy`

特点：

- 暖色编辑感
- 橙米色标签
- 大白卡片
- 留白充足

### `xhs-prd-ref`

这套风格来源于一次视觉风格提炼，只参考视觉，不参考原帖内容。

原始视觉提示词：

```text
小红书竖版图文视觉风格，深藏蓝背景，大面积留白与对齐，亮黄色圆角胶囊标签，白色悬浮卡片，卡片分区清晰，强结构化排版，标题大字高对比，扁平化设计，干净克制，理性清晰，适合移动端阅读，不参考原帖内容
```

### 风格使用规则

- 一篇帖子只允许一种风格
- 封面图和内容图必须统一
- 如果使用随机，只在 `editorial-airy` 和 `xhs-prd-ref` 中二选一
- 随机只发生一次，不能每一页单独随机

## 10. 图卡尺寸规则

当前输出比例已固定为：

- `1080 x 1440`
- `3:4` 竖版

这样做的目的：

- 更接近你当前参考图的视觉比例
- 让封面区与双分格区不会过长
- 提高在移动端连续滑动浏览时的稳定性

## 11. 发布机制说明

当标题、正文、PNG 图片都准备好之后，发布流程如下：

1. 保留原始输出在 `output/xhs-<topic>-<YYYYMMDD>/`
2. 复制 PNG 副本到 `/tmp/openclaw/uploads/...`
3. 打开小红书创作中心发布页
4. 如果默认落在视频发布，先切换到 `上传图文`
5. 上传 PNG 图片
6. 填入标题
7. 填入富文本正文
8. 点击 `发布`
9. 检查成功页
10. 检查发帖请求状态

### 登录态失败的判断

如果出现下面任意情况，必须停止并反馈：

- 跳到 `login?redirectReason=401`
- 出现二维码轮询接口，说明仍需扫码确认

### 成功判据

只有同时满足下面两点，才算真正发布成功：

- URL 含 `/publish/success`
- `web_api/sns/v2/note` 返回 `200`

## 12. 环境要求

要让这项技能能真正完成自动发布，必须满足这些前提：

- OpenClaw 可用
- 本机浏览器可被接管
- 小红书创作中心处于可用登录态
- Chrome / 浏览器自动化权限已授权
- 发布页面能正常打开并上传图文

如果这些条件不满足，技能仍可完成研究、文案和图片生成，但无法真实发布。

## 14. 排障建议

### 问题 1：能出图，但不能发布

优先检查：

- 小红书是否已登录
- 发布页是否跳到登录页
- 是否触发二维码扫码确认

### 问题 2：图有生成，但比例不对

优先检查：

- 是否使用了当前版本的 `generate_xhs_cards.py`
- 输出图片是否为 `1080 x 1440`
- 是否仍在使用旧的缓存图

### 问题 3：封面和内容页风格不一致

优先检查：

- `style-selection.json` 是否存在
- 是否在整篇里只选中一个 preset
- 是否误手动改了 `spec.json`

### 问题 4：研究内容太偏小红书站内

优先检查：

- 是否覆盖了 GitHub、知乎、36 氪、公众号、官方文档等来源
- 是否把小红书当成唯一信号源

## 16. 这项技能的价值

这项技能真正有价值的地方，不在于“会写一段文案”，而在于它把下面这些步骤连成了一条稳定流程：

- 研究
- 提炼
- 结构化
- 生图
- 文案
- 发布
- 成功校验

也就是说，它更像一个“小红书内容生产流水线技能”，而不是单点能力。

## 17. 总结

如果你要在 GitHub 上介绍这项技能，可以把它理解成：

> 一个面向 OpenClaw / Codex 的通用小红书研究与发布技能。输入一个题材方向，它会先全网研究，再整理成适合小红书的内容结构，自动生成统一风格的封面图与内容图，最后接管已登录的小红书创作中心完成图文发布并校验结果。
