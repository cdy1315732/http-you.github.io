# 小红书研究与发布流程

## 何时使用

当用户出现下面这类需求时，使用这个技能：

- 给一个题材方向，让你先去网上搜集火热内容，再做成小红书
- 围绕一个方向，整理热门观点 / 热门场景 / 热门技巧 / 热门推荐
- 生成封面图、内容图、标题、正文，然后直接发到小红书
- 希望你执行完整流程：研究 -> 文案 -> 配图 -> 发布

## 核心流程

### 第一步：接收题材方向

把用户输入当成“方向”，不是最终标题。

例如：
- AI 副业
- OpenClaw 使用技巧
- 企业自动化工作流
- 本地部署模型怎么选
- 个人效率工具

你的第一目标不是立刻写文案，而是先把这个方向拆成值得发的小红书角度。

### 第二步：联网收集热门内容

必须先搜索，再写内容。

优先顺序：
1. 小红书站内搜索页
2. 官方文档 / 官方产品页 / 官方公告
3. 高质量教程、厂商指南、社区帖子

搜索时重点收集：
- 反复出现的热门场景
- 反复出现的热门痛点
- 反复出现的热门技巧
- 反复出现的避坑点
- 反复出现的推荐清单

要求：
- 优先找多个来源反复提到的内容
- 不要只拿单一来源的观点直接下结论
- 把研究结论保存到 `research-notes.md`

### 第三步：整理成适合小红书的结构

根据题材决定结构：
- 10条：适合场景、技巧、错误、推荐、工具清单
- 5条：适合模型、技能、方案、对比类选题
- 3段式：适合 SOP、教程、策略类选题

默认图文结构：
- 1 张封面图
- 5 张内容图
- 每张内容图放 2 个点

默认正文结构：
- 开头一句话说明为什么值得看
- 编号列出核心内容
- 最后补 2 到 3 条结论
- 加 2 到 3 个话题标签

### 第四步：生成封面图和内容图

使用 `scripts/generate_xhs_cards.py` 生成图卡。

要求：
- 封面图必须短、直、抓眼球
- 内容图必须适合手机竖屏阅读
- 每个内容块至少回答一个问题：
  - 怎么做
  - 为什么火
  - 优点
  - 劣势
  - 适合谁
  - 避坑点

输出目录建议：
- `output/xhs-<topic>-<YYYYMMDD>/`

目录里至少包含：
- `01-cover-*.svg(.png)`
- `02-*.svg(.png)` 到 `06-*.svg(.png)`
- `post.md`
- `research-notes.md`
- `spec.json`

### 第五步：生成标题和正文

写成小红书风格，不写成长文章。

规则：
- 句子短
- 开头直接切重点
- 中间用编号或卡片感结构
- 少抽象解释，多直接表达
- 标题尽量短，避免平台截断

### 第六步：登录并发布到小红书

使用浏览器 profile：`xhs`

发布流程：
1. 打开 `https://creator.xiaohongshu.com/publish/publish`
2. 切换到 `上传图文`
3. 把 PNG 素材复制到 `/tmp/openclaw/uploads/...`
4. 上传图片
5. 用 DOM 填标题和正文
6. 点击 `发布`
7. 用下面两个条件一起确认成功：
   - URL 包含 `/publish/success`
   - `POST https://edith.xiaohongshu.com/web_api/sns/v2/note` 返回 `200`

如果失败：
- 如果跳到登录页，说明登录态过期
- 如果出现二维码轮询接口，说明还需要重新扫码
- 没有成功页和 `note` 接口 200，不允许声称“已发布成功”

## 固定原则

- 题材每次都可以不同
- 流程每次都一样
- 一定先研究，再文案，再配图，再发布
- 发布是最后一步，不是第一步

## 文件约定

默认输出目录：
- `output/xhs-<topic>-<YYYYMMDD>/`

默认文件：
- `01-cover-*.svg(.png)`
- `02-*.svg(.png)` 到 `06-*.svg(.png)`
- `post.md`
- `research-notes.md`
- `spec.json`
